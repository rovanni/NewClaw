1. O que é a Análise Léxica? (O Filtro)

Imagine que o código fonte é uma sopa de letras. O Scanner é como uma peneira que separa o que presta do que é lixo.

As 4 Tarefas do Scanner:
  1. Leitura de Caracteres: Lê o arquivo um por um (ex: 'i', depois 'n', depois 't').
  2. Agrupamento (Lexemas): Junta 'i'+'n'+'t' e reconhece a palavra "int".
  3. Classificação (Tokens): Diz para o computador: "Isso é um Tipo de Dado Inteiro".
  4. Limpeza: Remove espaços, tabs e comentários que não servem para o computador.
Slide 4

Visualizando a "Esteira de Produção"

Veja como uma linha simples de código é "mastigada" pelo compilador:

Entrada (Texto): int x = 10 + 20;
↓ (O Scanner entra em ação)
Resultado do Scanner (Tokens):
[T_INT] [T_ID:x] [T_ASSIGN] [T_NUM:10] [T_PLUS] [T_NUM:20]
↓ (O Parser recebe os Tokens)
Resultado do Parser (Lógica):
"Atribua à variável 'x' o resultado da soma entre 10 e 20"

O Scanner simplifica o mundo para o resto do compilador.

Slide 5

2. Por que o Scanner é importante para o Parser?

Imagine que o Parser (Sintático) é um juiz. O Scanner é o assistente que organiza os papéis.

O que acontece nos bastidores:
  • Redução de Complexidade: O Parser não precisa lidar com "espaços em branco" ou "caracteres individuais". Ele só vê Tokens prontos.
  • Eficiência: É muito mais rápido para o computador ler uma lista de 10 tokens do que processar 100 caracteres soltos.
  • Tratamento de Erros: Se o usuário digitar um caractere proibido (ex: '@'), o Scanner barra na hora!

4. Por que não pular a Análise Léxica?

Poderíamos ler tudo de uma vez? Sim, mas seria um pesadelo técnico.

Vantagens da Separação:
  1. Manutenção: Se você quiser mudar o símbolo de atribuição de = para :=, você só mexe no Scanner.
  2. Portabilidade: Facilita ler arquivos de diferentes sistemas operacionais (Windows vs Linux) que usam quebras de linha diferentes.
  3. Gramática Limpa: O Parser não precisa se preocupar com onde o programador colocou um espaço em branco.
O léxico é a "higienização" necessária para a segurança do compilador.

8. Expressões Regulares (ER): A Linguagem das Regras

Como explicamos para o C++ o que é um "Número" ou um "Nome"? Usamos ERs.

  • Números: [0-9]+ (Significa: Pelo menos um dígito de 0 a 9).
  • Identificadores: [a-zA-Z_][a-zA-Z0-9_]* (Significa: Começa com letra ou _, depois pode ter qualquer coisa).
  • Operadores: Símbolos fixos como +, -, *, /, =.
Pense na ER como um "Formulário de Validação" para cada palavra do código.

12. O Modelo de Estados (O "GPS" do Scanner)

Um autômato é como um GPS: dependendo de onde você está e qual símbolo você lê, ele te leva para um novo lugar.

  • Estado Inicial (q0): O ponto de partida. A máquina está "limpa", pronta para ler o primeiro caractere.
  • Transições: Se eu ler '1', vou para o caminho A. Se eu ler 'a', vou para o caminho B.
  • Estado de Aceitação (Fim): Quando a palavra acaba e a máquina diz: "Sucesso! Encontrei um Token".
Slide 18

13. Exemplo Real: Como reconhecer um Número?

Imagine que o Scanner está lendo "123;". Como ele sabe que "123" é um número?

  1. Lê '1': "Opa, é um dígito. Estou no estado 'Lendo Número'".
  2. Lê '2': "Outro dígito. Vou acumulando... 12".
  3. Lê '3': "Mais um! Acumulando... 123".
  4. Lê ';': "Espera! Isso NÃO é um dígito. O número acabou em 123!".
O scanner "devolve" o ';' para a próxima leitura e entrega o Token de Número.
Slide 19

14. O Dilema da "Espiadinha" (Lookahead)

Como o Scanner diferencia < (Menor) de <= (Menor ou Igual)?

A Técnica do Lookahead:

  1. Lê o caractere <.
  2. A máquina para e "olha" o próximo caractere sem sair do lugar.
  3. Se o próximo for =, ela consome os dois e retorna T_LE (Less Equal).
  4. Se não for =, ela retorna apenas T_LT (Less Than).

15. Nomes vs Comandos (Palavras Reservadas)

Por que if é um comando e idade é uma variável?

Ambos seguem a mesma regra de "Identificador". O segredo está na Tabela de Palavras Reservadas:

1. Scanner lê "if". 2. Ele classifica como ID (Identificador) temporariamente. 3. Ele olha em uma tabela: "O texto 'if' está na lista de comandos?". 4. SIM! Então ele troca o tipo para T_IF.

Se não estiver na lista, ele continua sendo apenas um T_ID (Nome de variável).

17. Como o peek() funciona na prática

O segredo da "espiada" sem movimento é essencial para alunos com dificuldade:

Código: int x == 10; ^ ponteiro 'pos'
  • peek(): Funciona como olhar através de uma fresta. Você vê o que está na frente (o próximo '='), mas seus pés (o ponteiro pos) continuam no mesmo lugar.
  • next(): Você olha o caractere E dá um passo à frente. Agora seus pés estão no novo caractere.
Lógica em C++:
if (peek() == '=') { next(); return T_EQ; }

20. O Coração do Compilador: nextToken()

Esta é a função mais importante. Ela é o "motor" que gira a engrenagem:

Token nextToken() { skipWhitespace(); // 1. Limpa a sujeira char c = peek(); // 2. Olha o que vem if (isdigit(c)) return scanNumber(); // 3. Se for número, trata if (isalpha(c)) return scanIdentifier(); // 4. Se for letra, trata switch(c) { case '+': next(); return Token(T_PLUS); // 5. Símbolos fixos // ... outros casos } }
Slide 27

21. Passo a Passo: scanNumber()

Como a função de ler números funciona internamente?

  1. Cria uma string vazia buffer.
  2. Inicia um loop: while (isdigit(peek())).
  3. Dentro do loop: buffer += next(); (Consome e guarda).
  4. Quando achar algo que não é número, sai do loop.
  5. Retorna um Token do tipo T_NUM com o valor guardado.
Slide 28

22. Passo a Passo: scanIdentifier()

Quase igual à leitura de números, mas com uma regra extra:

  1. Lê todas as letras e números (ex: "while").
  2. Antes de retornar, consulta a Tabela de Reservadas.
  3. Se for "while", retorna T_WHILE.
  4. Se for "x", retorna T_ID.
Lembre-se: O Scanner é o primeiro a saber se uma palavra é um comando ou apenas um nome.

O que acontece se o programador digitar o caractere $ dentro de um nome de variável no MicroC?

Opção A: O Scanner ignora e segue em frente.
Opção B: O Scanner avisa um "Erro Léxico".

Por que? No MicroC, o símbolo $ não faz parte do alfabeto permitido. O Scanner é o segurança que barra intrusos.

23. Limpando a Sujeira: Espaços e Comentários

Como o compilador "esquece" o que não importa?

void skipWhitespace() { while (isspace(peek()) || peek() == '/') { if (peek() == '/') { if (peekNext() == '/') { // Comentário de linha // while (peek() != '\n' && !isEOF()) next(); } else return; // Pode ser um operador de divisão } else next(); // Pula espaço, tab, \n } }
Pular espaços é o que permite que você escreva o código com a indentação que quiser.
Slide 31

24. O Sentinela: Como saber que o arquivo acabou?

O Scanner não pode ler além do fim do arquivo, senão o programa trava!

  • Usamos o caractere EOF (End of File).
  • No C++, verificamos se o índice atingiu o code.length().
  • Sempre que a máquina chega no fim, ela emite um Token(T_EOF).
Dica: Sem o Token de Fim, o Parser ficaria esperando código para sempre!

25. Performance: O Buffer Duplo (Curiosidade)

Compiladores profissionais (como o do GCC) não leem um caractere por vez do disco.

Eles carregam um "bloco" de 4096 caracteres na memória de uma vez. Isso é 1000x mais rápido do que ler do HD bit a bit.

No MicroC, usaremos std::string para simplificar, mas saiba que no mundo real a performance é levada a sério.

Slide 34

27. Determinismo (AFD) vs Não-Determinismo (AFN)

Por que convertemos tudo para AFD?

  • AFN: Pode estar em vários estados ao mesmo tempo (confuso para o computador).
  • AFD: Tem um caminho único e claro (perfeito para o comando switch do C++).

Todo Scanner eficiente é um AFD.

Slide 40

Passo 2: A Estrutura do Token

Crie a struct que carregará a informação de cada palavra:

struct Token { TokenType type; string lexeme; int line; };
Slide 41

Passo 3: A Classe Scanner (O Esqueleto)

class Scanner { string source; int pos = 0; int line = 1; public: Scanner(string s) : source(s) {} char peek() { return (pos < source.length()) ? source[pos] : '\0'; } char next() { return source[pos++]; } bool isEOF() { return pos >= source.length(); } Token nextToken(); // Vamos implementar a seguir! };
Slide 42

Passo 4: Implementando o Coração (nextToken)

Token Scanner::nextToken() { skipWhitespace(); if (isEOF()) return {TokenType::T_EOF, "", line}; char c = peek(); if (isdigit(c)) return scanNumber(); if (isalpha(c)) return scanIdentifier(); switch(c) { case '+': next(); return {TokenType::T_PLUS, "+", line}; case '=': next(); return {TokenType::T_ASSIGN, "=", line}; case ';': next(); return {TokenType::T_SEMICOLON, ";", line}; default: return {TokenType::T_ERROR, string(1, next()), line}; } }
Slide 43

Passo 5: Como testar seu Scanner?

No main(), crie um loop para imprimir o que o Scanner encontrou:

int main() { Scanner sc("int x = 10 + 20;"); Token t; while ((t = sc.nextToken()).type != TokenType::T_EOF) { cout << "Token: " << t.lexeme << " [Tipo: " << (int)t.type << "]" << endl; } return 0; }

Dicas de Ouro para o Laboratório

  • Cuidado com o loop infinito: Se você esquecer de chamar next(), o Scanner ficará lendo o mesmo caractere para sempre.
  • Use o cout para debugar: Imprima o caractere que o Scanner está lendo em cada passo.
  • Teste aos poucos: Primeiro faça funcionar apenas números, depois operadores.
