🧭 Tutorial do NewClaw

Guia passo a passo para configurar, conversar, auditar e cuidar da memória do sistema.

1. ConfigProvider, modelo, idioma e tools 2. ChatConverse, use voz, exporte conversas 3. TracesAudite decisões, tools e respostas 4. MemóriaExplore o grafo, revise e conecte nós
Se o sistema acabou de ser instalado, comece por Config antes de testar qualquer tool.

🚀 Passo 1 — Configurar o Sistema

Abra Config e configure:

  • Provider — escolha Ollama, Gemini, DeepSeek ou Groq
  • Modelo — selecione ou digite o modelo (ex: glm-5.1:cloud)
  • Idioma — ajuste o idioma preferencial
  • Tools — ative/desative ferramentas conforme necessidade
  • Salvar & Reiniciar — aplique mudanças de provider ou modelo
Fluxo recomendado: selecione provider → ajuste modelo → confira lista de modelos → salve.

💬 Passo 2 — Conversar no Chat

Na tela Chat:

  • + Nova — cria conversa vazia
  • 🎤 — reconhecimento de voz do navegador
  • 🔊 — liga/desliga TTS automático
  • 📥 / 📤 — exporta/importa conversas em JSON
  • 📋 — copia resposta individual (hover na mensagem)
Enter envia, Shift+Enter cria nova linha.

🔍 Passo 3 — Auditar com Traces

Na tela Traces, veja o caminho completo de cada execução:

  • Decisão — como o agente classificou a intenção
  • Tool Calls — quais ferramentas foram chamadas
  • Resultados — o que cada tool retornou
  • Resposta Final — texto enviado ao usuário
Se o chat respondeu errado, olhe o trace antes de mexer no prompt.
4

Memória Textual

Na tela Memória:

  • Buscar — procura por nome ou conteúdo
  • Filtros — separa por tipo (identity, fact, skill...)
  • Salvar — atualiza nó selecionado
  • Deletar — remove nó e relações
  • Adicionar Relação — conecta dois nós
5

Grafo de Memória

Na tela Grafo:

  • ➕ Novo Nó — cria nó manualmente
  • 🔗 Conectar — liga dois nós visualmente
  • 📷 PNG / SVG / JSON — exporta o grafo
  • 🔍 Busca — busca semântica no grafo
  • 📊 Métricas — PageRank, Louvain, comunidades
  • Clique num nó → painel direito com detalhes e edição
🔍 Modo Foco: Selecione um nó → clique "Isolar Nó" para ver apenas ele e vizinhos. Use profundidade (1 hop / 2 hops) e o slider de importância para filtrar ruído.
6

Revisão de Memória

Na tela Revisão:

  • Mesclar — resolve duplicatas
  • Interligar Datas — cria relações cronológicas
  • Órfãos — nós sem conexões
  • Nós Fracos — baixo pagerank
Use mescla apenas quando os pares representam o mesmo conceito.
🛠

Ferramentas do Agente (8)

  • exec_command ⚠️ — executa comandos shell
  • web_search 🔍 — busca iterativa com leitura de páginas e síntese multi-fonte
  • file_ops 📁 — lê, grava e lista arquivos
  • memory_search 🧠 — busca semântica na memória
  • memory_write ✏️ — create, update, connect, delete, merge
  • send_audio 🔊 — TTS e envio de voz no Telegram
  • send_document 📄 — envia arquivos (HTML, PDF, imagens)
  • memory_admin 🔧 — diagnosticar, limpar, reindexar

🔍 Modo Foco — Navegação Inteligente

O grafo cognitivo pode ter centenas de nós. O Modo Foco resolve o problema de "tudo amontuado":

1. Isolar Nó

Clique em qualquer nó → clique 🔍 Isolar Nó na sidebar. Apenas o nó selecionado e seus vizinhos diretos ficam visíveis. O resto fica esmaecido.

2. Profundidade

1 hop — mostra apenas conexões diretas (recomendado).
2 hops — inclui vizinhos dos vizinhos.
Tudo — mostra o grafo completo.

3. Importância

O slider ⭐ Importância mín. filtra nós com PageRank baixo. Aumente para ver apenas os nós mais relevantes.

4. Visão Geral

Clique 🔄 Visão Geral ou clique no fundo do grafo para restaurar a visualização completa.

Dica: Comece com 1 hop e slider em 0%. Ajuste conforme necessário. O objetivo é ver contexto relevante, não o grafo inteiro.

❓ Dúvidas Comuns

O chat respondeu errado. → Abra Traces, verifique a decisão e tool chamada antes de mexer no prompt.

Troquei o modelo e nada mudou. → Use Salvar & Reiniciar na tela Config.

A memória está bagunçada. → Use memory_admin com ação cleanup para remover ghosts e órfãos automaticamente. No dashboard: Grafo → Modo Foco para visualizar sem ruído.

Posso usar só pelo Telegram? → Sim, mas o dashboard é essencial para config, traces e manutenção.
