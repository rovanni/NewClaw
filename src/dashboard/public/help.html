Capítulo 1 — Primeira Execução

Tire o sistema do zero. Entenda o que é, configure e salve.

▼

O que é o NewClaw?

NewClaw é um agente de IA com memória persistente e ferramentas reais. Diferente de chatbots comuns, ele lembra de você entre conversas e pode executar ações: buscar na web, ler e gravar arquivos, administrar sua própria memória, enviar áudios e documentos.

O que ele faz diferente?

✅ O que esperar

  • Memória entre sessões
  • Execução de ferramentas reais
  • Auditoria completa (Traces)
  • Grafo cognitivo visual

❌ O que NÃO esperar

  • Chat comum sem contexto
  • Respostas genéricas sem fundamento
  • "Esquecer" tudo ao recarregar
  • Ações sem rastreabilidade

Configuração passo a passo

  1. Abra a tela Config no menu superior
  2. No campo Provider, escolha seu provedor de IA (Ollama, Gemini, DeepSeek ou Groq)
  3. No campo Modelo, selecione ou digite o modelo desejado (ex: glm-5.1:cloud)
  4. Ajuste o Idioma para pt-BR
  5. Confira as Tools ativas — por padrão, todas estão habilitadas
  6. Clique Salvar & Reiniciar para aplicar as mudanças

✓ Resultado: Sistema configurado e pronto para a primeira conversa. O agente já pode responder com o modelo escolhido.

⚠️ Se o sistema acabou de ser instalado, comece SEMPRE por aqui. Pular este passo = respostas vazias ou erros de modelo.

Capítulo 2 — Primeira Conversa Útil

Gere valor rápido. Aprenda a conversar, usar voz e exportar.

▼

Conversa guiada

Vamos fazer uma conversa real para ver o sistema em ação. Abra o Chat e siga:

Exemplo prático

Digite no chat:

Me ajude a organizar meus estudos desta semana

Observe a resposta — o agente pode:

  • Buscar na memória informações sobre você
  • Usar memory_search para encontrar contexto anterior
  • Estruturar uma resposta personalizada

Controles do Chat

🎤 Voz

🎤 — Ativa reconhecimento de voz do navegador (fale e o texto aparece)
🔊 — Liga/desliga TTS automático (o agente lê as respostas em voz alta)

📂 Exportar / Importar

📥 — Exporta a conversa em JSON
📤 — Importa conversa salva
📋 — Copia resposta individual (hover na mensagem)

Gerenciar conversas

  1. Clique + Nova para criar uma conversa vazia
  2. Digite sua mensagem e pressione Enter
  3. Use Shift+Enter para criar nova linha sem enviar
  4. No histórico à esquerda, troque entre conversas
  5. Para exportar: clique 📥 e salve o arquivo JSON

💡 Dica: O agente já está usando a memória! Se você disser "Meu nome é Luciano" e depois perguntar "Qual é meu nome?", ele saberá — mesmo em conversas futuras.

Capítulo 3 — Entender o Cérebro (Traces)

Veja COMO o agente pensa. Decisão → Tools → Resposta.

▼

Por que Traces importam?

Quando o agente responde, ele faz um caminho invisível: classifica a intenção, decide se usa ferramentas, executa, e sintetiza a resposta. O Traces revela esse caminho. Isso cria confiança técnica — você sabe exatamente o que aconteceu.

Como ler um Trace

Estrutura de um trace 
1. Decisão    → "O usuário quer buscar informação"
2. Tool Call  → web_search("tendências de IA 2025")
3. Resultado  → [3 fontes encontradas e sintetizadas]
4. Resposta   → "As principais tendências são..."

Passo a passo

  1. Abra a tela Traces
  2. Selecione uma conversa no dropdown
  3. Clique em qualquer mensagem para expandir o trace
  4. Verifique: a Decisão fez sentido? As Tool Calls foram corretas?
  5. Se algo saiu errado, o trace mostra exatamente onde quebrou

🔧 Troubleshooting: Se o chat respondeu errado, não mude o prompt primeiro. Abra o Traces e veja onde o agente tomou a decisão errada. Corrija a causa, não o sintoma.

Capítulo 4 — Memória (O Diferencial)

Entenda o que são nós, tipos e como editar. Aqui o sistema brilha.

▼

O que é a memória?

A memória é um conjunto de nós — unidades de conhecimento que o agente armazena e consulta. Cada nó tem um tipo, um conteúdo e conexões com outros nós. É isso que faz o agente "lembrar" de você.

Tipos de nó

🆔 Identity

Quem você é. Nome, profissão, preferências. O núcleo da memória.

📌 Fact

Fatos e informações. "Luciano é professor na UTFPR."

⚡ Skill

Capacidades e habilidades. "Sabe Python e Docker."

🔗 Relation

Conexões entre nós. "Luciano → trabalha_em → UTFPR".

Exemplo prático — Criar um nó

Mão na massa
  1. Abra a tela Memória
  2. Clique + Novo Nó (ou use o campo de busca e depois "Adicionar")
  3. Preencha: Tipo = identity, Nome = Luciano, Conteúdo = Professor de Engenharia de Software na UTFPR
  4. Clique Salvar

Teste a memória no chat

Verifique se funcionou
O que você sabe sobre mim?

O agente deve responder incluindo a informação que você acabou de salvar. Se não incluir, verifique no Traces se memory_search foi chamado.

Editar e conectar nós

  1. Na tela Memória, clique em qualquer nó para editar
  2. Use Adicionar Relação para conectar dois nós (ex: "Luciano → ensina → Engenharia de Software")
  3. Use Filtros para ver apenas um tipo (identity, fact, skill...)
  4. Deletar remove o nó e todas as suas relações

💡 Dica: Quanto mais rico o grafo de identidade, mais personalizada será cada resposta. Invista 5 minutos preenchendo seus dados — o retorno é imediato.

Capítulo 5 — Grafo Cognitivo (Visualização)

Navegue, isole nós, entenda relações. O mapa vivo do conhecimento.

▼

O que é o grafo?

O Grafo é a representação visual de toda a memória. Cada bolha é um nó, cada linha é uma relação. É como um mapa mental que cresce conforme você conversa com o agente.

Navegação básica

  1. Abra a tela Grafo
  2. Arraste para mover, scroll para zoom, clique num nó para selecionar
  3. O painel direito mostra detalhes: nome, tipo, conteúdo, conexões
  4. Clique ➕ Novo Nó para criar direto no grafo
  5. Clique 🔗 Conectar para ligar dois nós visualmente

Modo Foco — Navegação inteligente

Quando o grafo tem centenas de nós, tudo fica amontoado. O Modo Foco resolve isso:

🔍 Isolar Nó

Selecione um nó → clique Isolar Nó. Apenas ele e vizinhos ficam visíveis. O resto fica esmaecido.

📏 Profundidade

1 hop = conexões diretas (recomendado).
2 hops = vizinhos dos vizinhos.
Tudo = grafo completo.

⭐ Importância

O slider filtra nós com PageRank baixo. Aumente para ver apenas os mais relevantes.

🔄 Visão Geral

Clique Visão Geral ou clique no fundo do grafo para restaurar a visualização completa.

Exportar o grafo

  1. 📷 PNG — imagem estática para compartilhar
  2. 📷 SVG — vetor editável (illustrator, figma)
  3. 📄 JSON — dados completos para backup ou integração

Métricas avançadas

Clique 📊 Métricas para ver PageRank (importância de cada nó) e comunidades (grupos naturais detectados pelo algoritmo Louvain). Nós com PageRank alto são os mais "centrais" no conhecimento.

💡 Dica: Comece com 1 hop e slider em 0%. Ajuste conforme necessário. O objetivo é ver contexto relevante, não o grafo inteiro.

Capítulo 6 — Ferramentas (Poder Real)

Exec_command, web_search, memory_write — com exemplos práticos.

▼

O que são as ferramentas?

Ferramentas são ações reais que o agente pode executar durante uma conversa. Ele decide quando usar qual ferramenta — mas você pode solicitar explicitamente.

As 8 ferramentas

⚡ exec_command

Executa comandos shell. ⚠️ Perigoso — use com cuidado.

🔍 web_search

Busca iterativa na web, lê páginas e sintetiza multi-fonte.

📁 file_ops

Lê, grava e lista arquivos no servidor.

🧠 memory_search

Busca semântica na memória do agente.

✏️ memory_write

Cria, atualiza, conecta, deleta e mescla nós de memória.

🔊 send_audio

Gera TTS e envia voz no Telegram.

📄 send_document

Envia arquivos (HTML, PDF, imagens) pelo Telegram.

🔧 memory_admin

Diagnosticar, limpar, reindexar a memória.

Exemplo 1 — Buscar e salvar

Comando no chat
Busque as tendências de IA para 2025 e salve como memória

O agente vai: web_search → coletar fontes → memory_write → salvar nó tipo fact
Depois, você pode perguntar "Quais tendências de IA você conhece?" e ele vai usar a memória.

Exemplo 2 — Executar e auditar

Comando no chat
Execute ls -la /home e me mostre o resultado

O agente vai: exec_command → mostrar resultado.
Depois, abra Traces para ver a chamada completa e o retorno.

Exemplo 3 — Criar nó via chat

Comando no chat
Salve na memória: Prefiro respostas em português e formato markdown

O agente cria um nó tipo preference automaticamente e já passa a considerar isso nas próximas respostas.

⚠️ Segurança: exec_command executa comandos reais no servidor. Ative apenas se confiar no agente e no modelo. Desative em Config → Tools se não precisar.

Capítulo 7 — Usuário Avançado

Revisão de memória, merge, PageRank, limpeza e manutenção.

▼

Revisão de Memória

A tela Revisão é sua ferramenta de curadoria. Com o tempo, a memória acumula duplicatas, nós órfãos e informações desatualizadas. A revisão resolve isso.

🔀 Mesclar

Resolve duplicatas. Quando dois nós representam o mesmo conceito, mescle-os em um só. O conteúdo do nó mesclado combina ambos.

📅 Interligar Datas

Cria relações cronológicas automáticas entre nós que mencionam datas. Útil para reconstruir timelines.

👻 Órfãos

Nós sem nenhuma conexão. Podem ser removidos com segurança ou conectados manualmente.

📉 Nós Fracos

Nós com PageRank baixo — pouco relevantes para o grafo. Candidate-se para limpeza.

Manutenção com memory_admin

Comandos no chat
Diagnostique a memória e me diga se tem problemas

O agente usa memory_admin com ação diagnose para relatório completo.

Limpe a memória — remova ghosts e órfãos

O agente usa memory_admin com ação cleanup para remover automaticamente.

Fluxo de manutenção recomendado

  1. Semanalmente: Abra Revisão → mescle duplicatas
  2. Quinzenalmente: Use memory_admin cleanup no chat
  3. Mensalmente: Exporte o grafo em JSON (backup) e revise nós fracos
  4. Quando necessário: Interligar datas para reconstruir timelines

✓ Parabéns! Se você chegou até aqui, domina o NewClaw. O sistema está configurado, a memória está curada, e você sabe auditar cada decisão do agente.

❓ Dúvidas Comuns

  • O chat respondeu errado. → Abra Traces, verifique a decisão e tool chamada antes de mexer no prompt.
  • Troquei o modelo e nada mudou. → Use Salvar & Reiniciar na tela Config.
  • A memória está bagunçada. → Use memory_admin com ação cleanup. No dashboard: Grafo → Modo Foco para visualizar sem ruído.
  • Posso usar só pelo Telegram? → Sim, mas o dashboard é essencial para config, traces e manutenção.
  • Como reverter uma mudança na memória? → Exporte o grafo em JSON regularmente. Para restaurar, importe o backup.
